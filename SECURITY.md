# Security and Safety

If you discover a security vulnerability or sensitive data exposure, please report it responsibly.

- Email: security@example.com (replace with your contact) or open an issue in the repository.
- For urgent disclosures involving minors or personally identifiable information (PII), contact the project owner immediately.

Content safety and minors:
- The application is intended for educational use. Do not collect or store any personally identifying information about minors without explicit parental consent and secure data handling.
- If you deploy a public instance, review privacy laws (e.g., COPPA, GDPR) and configure the app to avoid storing PII. Use anonymous identifiers and secure logging.

Operational guidance:
- Use secure secrets management (Streamlit Cloud secrets, GitHub Secrets) for API keys or credentials.
- Limit repository admin access and set branch protection rules.
